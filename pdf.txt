PROYECTO 1
EJECUCION DEL PROMPT EN EL ASISTENTE DE SOPORTE TECNICO CON GEMINI

1. Introduccion

En este proyecto se implementa un asistente de soporte tecnico en Python que utiliza el modelo Gemini para analizar consultas de usuarios y responder en formato JSON. La ejecucion del prompt es el proceso central del sistema, ya que define como la inteligencia artificial interpreta el contexto y genera respuestas utiles, claras y estructuradas.

2. Objetivo de la ejecucion del prompt

El objetivo principal es enviar al modelo una instruccion completa que incluya:

- El rol del asistente (analista de soporte tecnico).
- El formato de salida obligatorio (JSON valido).
- Reglas de clasificacion por tema.
- Restricciones para evitar respuestas ambiguas.
- Historial conversacional para mantener continuidad.

Con esto, el sistema busca responder de manera consistente y orientada a la resolucion de problemas tecnicos reales.

3. Flujo de ejecucion

La ejecucion del prompt se desarrolla en las siguientes etapas:

3.1 Carga de configuracion

Al iniciar el script, se cargan variables de entorno desde el archivo .env. En esta fase se obtiene la clave GENAI_API_KEY, necesaria para autenticar la conexion con Gemini.

3.2 Inicializacion del cliente

Con la clave API disponible, se crea el cliente de Google GenAI. Este cliente sera el encargado de enviar prompts y recibir respuestas del modelo.

3.3 Construccion del prompt

Cuando el usuario escribe una consulta, el sistema construye un prompt compuesto por:

- Prompt de sistema: define tarea, formato JSON, catalogo de temas y reglas de respuesta.
- Historial: agrega interacciones anteriores para conservar contexto.
- Consulta actual: texto ingresado por el usuario en ese turno.

Esta construccion garantiza que cada nueva respuesta considere tanto la instruccion base como el estado actual de la conversacion.

3.4 Envio al modelo

El prompt completo se envia mediante el metodo de generacion de contenido usando el modelo gemini-2.5-flash.

3.5 Procesamiento de respuesta

La salida del modelo se valida y se presenta en consola. Si la respuesta llega en formato JSON correcto, se imprime con formato legible. Si ocurre un error de parseo, se muestra el texto recibido para facilitar diagnostico.

3.6 Manejo de errores

El sistema contempla errores frecuentes, por ejemplo:

- Clave API no configurada.
- Error 403 por permisos o credenciales invalidas.
- Clave API reportada como filtrada.

Cuando se detecta alguno de estos casos, el programa informa al usuario con un mensaje claro y propone accion correctiva.

3.7 Actualizacion del historial

Despues de cada intercambio, la consulta del usuario y la respuesta del asistente se agregan al historial para mantener continuidad en la siguiente interaccion. Adicionalmente, el historial se limita para evitar prompts demasiado extensos.

4. Resultado esperado

Como resultado de la ejecucion del prompt, el asistente genera respuestas estructuradas con esta logica:

- tema_principal: categoria tecnica detectada.
- estado: aceptado o requiere_aclaracion.
- respuesta: orientacion tecnica concreta o preguntas de seguimiento.

Este formato permite estandarizar la salida y facilita su uso posterior en otras interfaces o procesos.

5. Conclusiones

La ejecucion del prompt en este proyecto demuestra como una buena ingenieria de instrucciones mejora la calidad de las respuestas de IA. Al combinar reglas claras, contexto conversacional y validacion de salida, el asistente logra un comportamiento mas estable, util y alineado con necesidades de soporte tecnico.

6. Recomendaciones

- Mantener actualizado el prompt de sistema segun nuevos casos de uso.
- Validar siempre la estructura JSON de salida.
- Proteger la clave API con buenas practicas de seguridad.
- Registrar ejemplos de consultas reales para mejorar el rendimiento del asistente.
