## 🔹 1. Idea clave del texto

Las herramientas en pentesting **no son universales** → dependen del **objetivo que estás evaluando**.

👉 Primero defines **qué vas a atacar**, luego eliges **qué herramientas usar**.

---
## 🔹 2. Superficies de ataque (qué puedes evaluar)

- Infraestructura de red
- Redes inalámbricas (WiFi)
- Servidores web
- Bases de datos
- Sistemas operativos (Windows / Linux)
- Aplicaciones móviles
- APIs (backend)

---

## 🔹 3. Tipos de herramientas según el objetivo

### 🌐 3.1 Infraestructura de red

Sirven para analizar o atacar la red.

- Detectar tráfico (sniffing)
- Manipular tráfico
- Saturar dispositivos (DoS básico)
- Evadir firewalls o IPS

👉 Ejemplo mental: ver qué pasa dentro de la red y cómo romper sus defensas.

---

### 📡 3.2 Redes inalámbricas (WiFi)

Se enfocan en ataques sobre redes inalámbricas.

- Romper contraseñas (cracking)
- Desconectar usuarios (deauth attack)
- Ataques Man-in-the-Middle (interceptar comunicación)

👉 Aquí el objetivo es **entrar o espiar la red WiFi**.

---

### 🌍 3.3 Aplicaciones web

Muy importante (alto valor en la industria).

- Escáneres automáticos de vulnerabilidades
- Proxies de interceptación (para modificar peticiones)
- Pruebas manuales

👉 Aquí buscas fallos como:

- Inyección SQL
- XSS
- Fallos de autenticación

---

### 🗄️ 3.4 Bases de datos

Muchas veces se prueban junto con web.

- Detectar inyección SQL
- Acceso no autorizado
- Mala configuración

👉 Básicamente: **ver si puedes robar o modificar datos**.

---

### 💻 3.5 Sistemas (Windows / Linux)

Evaluas seguridad del sistema operativo.

- Escáneres de vulnerabilidades
- Detección de software desactualizado
- Configuraciones inseguras

👉 Ejemplo: servicios abiertos, permisos mal configurados.

---

### 📱 3.6 Aplicaciones móviles y APIs

Cada vez más importante.

- Análisis de apps móviles
- Pruebas de APIs backend
- Intercepción de tráfico móvil

👉 Aquí atacas:

- La app (frontend)
- El servidor (backend)

---

### 🔍 3.7 Fuzzing

Herramientas especiales.

- Envían datos aleatorios o malformados
- Buscan fallos o crashes

👉 Sirve para probar:

- Protocolos
- Software
- APIs

---

## 🔹 4. Resumen mental (muy importante)

Piensa así:

> 🔹 ¿Qué voy a atacar? → Red, web, móvil, sistema  
> 🔹 ¿Qué quiero lograr? → Detectar, explotar, interceptar  
> 🔹 ¿Qué herramienta necesito? → Depende de lo anterior

---

## 🔹 5. Punto crítico (esto te lo digo directo)

Si intentas aprender pentesting solo memorizando herramientas → vas a fallar.

Lo correcto es:

1. Entender **cómo funciona la tecnología**
2. Entender **cómo se rompe**
3. Luego aprender herramientas

## 🛠️Requisitos y Pautas

Debe configurar un laboratorio de práctica de pruebas de penetración porque algunas de las herramientas preferidas son nuevas para usted. ¿Qué procedimientos recomendados seguirá al configurar su laboratorio? 

1. Garantizar el acceso cerrado a la red e Internet.
2. Crear un entorno informático virtualizado.
3. Proporcionar suficientes recursos de hardware para garantizar resultados válidos.
